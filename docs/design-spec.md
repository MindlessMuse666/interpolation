## 1. Введение

Данный документ описывает визуальный язык, эстетику и правила оформления пользовательского интерфейса. Все указанные стили базируются на технологическом стеке проекта (Vuetify 3, CSS-переменные) и могут быть реализованы с помощью стандартных классов Material Design, дополненных кастомными темами и небольшими CSS-правилами.

---

## 2. Концепция дизайна

Эстетика интерфейса вдохновлена образами **Рэм и Рам** из аниме "Re:Zero" - пастельная розовая (Ram) и небесно-голубая (Rem) гамма. Дизайн сочетает мягкость учебной "песочницы" с аккуратными контрастными акцентами, обеспечивая:

- Дружелюбную, спокойную атмосферу для обучения;
- Чёткую иерархию и читаемость текста (WCAG AA);
- Ассоциативную привязку: розовый - актив, задания, ответы; голубой - теория, примеры, информация.

Лейтмотив - *"двойственный баланс"*: две стороны обучения (теория / практика), два ключевых персонажа, два основных цвета.

---

## 3. Цветовая палитра

| **Rem Blue (Secondary)** | `#5E9DC8` | Кнопки информации, подсказки, ссылки, график 2 |

### 3.2 Дополнительные оттенки

Для декоративных плашек, фонов графиков и состояний hover/active используются осветлённые варианты основных цветов:

- `Ram Pink Light` -> `#F5C6D3` (светло-розовый, фон для активных полей)
- `Rem Blue Light` -> `#C3E0F2` (светло-голубой, фон подсказок)
- `Surface Alt` -> `#F1F0F5` (чередование строк истории)

### 3.3 Контраст

Минимальный коэффициент контрастности для основного текста на Surface - **4.6:1**, на цветных кнопках (белый текст по Primary/Secondary) - **4.8:1**, что соответствует WCAG AA. Текст на светлых акцентных фонах (`Ram Pink Light`, `Rem Blue Light`) остаётся тёмно-серым для сохранения читаемости.

## 4. Типографика

### 4.1 Основной интерфейсный шрифт

В текущей реализации используется системный `sans-serif` (дефолт Vuetify/браузера). В теме Vuetify задано целевое семейство `"Inter", sans-serif`; при необходимости Inter подключается на уровне HTML/CSS.

### 4.2 Шрифт для чисел и формул

Для чисел и координат в формах используется моноширинный `monospace` (класс `math-font`). Для отображения формул подключён KaTeX (`katex/dist/katex.min.css`).

### 4.3 Размеры и иерархия

| Уровень | Размер (rem) | Вес | Применение |
|---------|--------------|-----|------------|
| H1 | 2.25rem (36px) | 700 (Bold) | Заголовок страницы |
| H2 | 1.75rem (28px) | 600 (SemiBold) | Основные разделы |
| H3 | 1.375rem (22px) | 600 | Заголовки карточек |
| Body Large | 1.125rem (18px) | 400 (Regular) | Основной текст, теория |
| Body | 1rem (16px) | 400 | Текст в карточках, задания |
| Caption | 0.875rem (14px) | 400 | Подсказки, метки, история |
| Math Input | 1rem (16px) | 500 (Medium) | Поля ввода чисел и координат (*monospace*) |

## 5. Компоненты и общие стили

### 5.1 Скругления (`border-radius`)

- **Карточки, панели:** `12px` (отступы внутри 16px-24px)
- **Кнопки:** `8px` (высота 40px для стандартных, минимум 44px для touch-устройств на мобильных)
- **Поля ввода:** `8px`
- **Модальные окна:** `16px`
- **Слайдеры/ползунки:** track - полностью закруглённый (pill), thumb - круглый
- **Графические контейнеры (canvas):** `border-radius: 12px; overflow: hidden;`

**Совет:** в Vuetify 3 глобально можно задать переменные `$border-radius-root` (например, 8px) и наследовать. Для карточек используйте класс `rounded-xl` (12px), для кнопок `rounded-lg` (8px).

### 5.2 Отступы и сетка

Используется 8px-сетка. Основные отступы:

- Между секциями: `24px` (1.5rem)
- Между блоками внутри секции: `16px`
- Внутренние отступы карточек: `20px 24px`
- Вертикальное ритмическое расстояние между строками текста 1.5.
- Контейнер страницы ограничен шириной `1200px` и выровнен по центру с паддингом `16px` (mobile) / `32px` (desktop).

### 5.3 Тени

Тени мягкие, с оттенком основного акцента, создают ощущение "парящего" интерфейса.

```css
--shadow-card: 0 4px 16px rgba(30, 41, 59, 0.07), 0 1px 3px rgba(30, 41, 59, 0.04);
--shadow-button: 0 2px 8px rgba(232, 140, 165, 0.25);
--shadow-focus: 0 0 0 3px rgba(94, 157, 200, 0.4);
```

Карточки используют `--shadow-card`. При наведении на кликабельные карточки тень немного усиливается (поднимается на 2px по Y).

### 5.4 Кнопки

- **Primary (Ram Pink):** сплошная заливка `#E88CA5`, текст белый, при наведении затемняется на 5%.
- **Secondary (Rem Blue):** сплошная заливка `#5E9DC8`, текст белый.
- **Tonal (менее выделенные):** фон `Ram Pink Light` с цветом текста `#E88CA5` (розовый по светлому), аналогично для голубого.
- **Outline:** прозрачный фон, рамка `1px solid #E88CA5` (или `#5E9DC8`), текст цвета рамки.
- **Danger/Error:** фон `#DC5A5A`, текст белый.
- **Текстовые кнопки:** цвет текста Primary/Secondary без заливки.

Все кнопки имеют `font-weight: 600`, `letter-spacing: 0.02em`, плавный переход фона `transition: background 0.2s ease`.

### 5.5 Поля ввода и формы

- Высота поля: `48px` для комфортного тапа.
- Фон: `#FFFFFF`, рамка `1px solid #E2E8F0`.
- При фокусе рамка меняется на `#E88CA5` (или `#5E9DC8` в зависимости от контекста), появляется тень `--shadow-focus`.
- Текст ввода - `JetBrains Mono` (чтобы числа и координаты выглядели аккуратно), остальные поля (поиск по истории) - `Inter`.
- Label и вспомогательный текст - `Inter`, размер `0.875rem`.
- Состояние ошибки: рамка `#DC5A5A`, под полем появляется текст ошибки красного цвета с иконкой.
- Placeholder - цвет `#94A3B8`.

### 5.6 Валидация

Стили валидации реализуются через классы Vuetify (`v-text-field` с `:rules`). Единый шаблон:

- Ошибка снизу: цвет `#DC5A5A`, иконка восклицательного знака в круге.
- Глобальное сообщение об ошибке (при отправке формы) - снек-бар с красной полосой слева.

### 5.7 Заголовки (Header) и навигация

**Header:**

- Высота `64px`, закреплён сверху (sticky).
- Фон с эффектом матового стекла: `backdrop-filter: blur(12px); background-color: rgba(248, 246, 249, 0.85);` чтобы сквозь него мягко просвечивал фон.
- Нижняя граница: `1px solid #E2E8F0`.
- Содержит название приложения (слева, вес 700, цвет `#1E293B`) и навигацию: две вкладки "Обучение" и "Песочница" в виде текстовых кнопок или пилюль подчёркивания текущего раздела.
- Активный пункт: подчёркивание цветом `#E88CA5` толщиной 2px снизу.

**Footer:**

- Высота `48px`, прижат к низу (или есть в потоке, если контента мало).
- Фон `#FFFFFF`, верхняя граница `1px solid #E2E8F0`.
- Текст: `0.75rem`, `#64748B`, ссылки на авторов/преподавателя.
- Паддинги по горизонтали такие же, как у основного контейнера.

### 5.8 Карточки (Card)

```css
.card {
  background: #FFFFFF;
  border-radius: 12px;
  box-shadow: var(--shadow-card);
  padding: 20px 24px;
  border: 1px solid #E2E8F0;
}
```

Все материалы теории, задания, панель истории, блок с графиком оборачиваются в такие карточки. Карточки не должны сливаться с фоном.

### 5.9 Графики (Chart.js)

- Контейнер графика имеет белый фон, скругление `12px`, тень как у карточки.
- Исходные точки (scatter): цвет `#E88CA5` (Ram Pink), форма - круг радиуса 5px, обводка отсутствует.
- Интерполяционная кривая: цвет `#5E9DC8` (Rem Blue), толщина линии 3px, без заливки под графиком (прозрачная область). Сглаживание (tension) по умолчанию 0.3, если требуется кусочно-линейная интерполяция - 0.
- Подписи осей: шрифт `Inter`, цвет `#64748B`, размер 12px.
- Сетка: горизонтальные и вертикальные линии `#E2E8F0` толщиной 1px, без tick-меток на верхней и правой осях.
- Легенда: при необходимости размещается сверху или справа, используется стиль с квадратными маркерами.

### 5.10 История вычислений

- Список последних 10 записей в боковой панели (overlay) или в нижней части страницы ("История").
- Каждая запись - строка с разделением: слева метод (цветной тег: розовый/голубой/серый), посередине краткое "f(1.5) = 2.75", справа кнопка повтора.
- При наведении строка подсвечивается фоном `#F1F0F5`.
- Кнопка повтора - иконка "стрелка по кругу", цвет `#5E9DC8`.

### 5.11 Заметки

- Используются `v-expansion-panels` в режиме accordion (несколько панелей могут быть раскрыты).
- Каждая панель выглядит как отдельная карточка: `border: 1px solid #E2E8F0`, `border-radius: 12px`, фон `#FFFFFF`.
- Между панелями используется отступ `12px` вместо системного разделителя.
- Визуальные разделители Vuetify между панелями отключаются (без толстой серой полосы).

### 5.12 PNG-иллюстрации

- PNG-изображения размещаются на Surface (`#FFFFFF`) внутри контейнера со скруглением `12px` и рамкой `1px solid #E2E8F0`.
- Иллюстрации выравниваются по центру и масштабируются по ширине карточки без искажений (с сохранением пропорций).
- Изображения должны быть уникальными для разных карточек и отражать смысл раздела.

## 6. Оформление основных страниц

### 6.1 Страница "Обучение" (`/learning`)

Состав:

1. **Теоретический блок** - карточки для каждой темы (Что такое интерполяция, Применение, Линейная, Лагранж, Ньютон). Внутри заголовок (H3), текст (Body Large), поясняющая PNG-иллюстрация по теме (без повторяющихся изображений между разными карточками). Акцентные слова внутри текста выделяются цветом `#E88CA5` с полужирным начертанием.
2. **Практика с подсказками** - отдельная карточка "Выполните задание". Содержит текст задачи, поле ввода ответа (JetBrains Mono) и кнопку "Проверить". При проверке ответ окрашивается в зелёный/красный. Под полем динамически появляется текст подсказки (Rem Blue Light фон, текст `#1E293B`).
3. **Свободная практика** - секция с упрощённой "песочницей": можно задать точки, выбрать метод, отправить запрос и сразу увидеть результат + график. Оформляется аналогично большой песочнице (см. ниже).

Для карточек теории используется композиция: текстовый блок, ниже - блок заметок (accordion), ниже - PNG-иллюстрация (если предусмотрена), выровненная по центру и масштабируемая по ширине карточки.

### 6.2 Страница "Песочница" (`/sandbox`)

Состав:

- **Управляющая панель** (карточка слева или сверху): поля ввода точек (динамический список строк x | y с кнопками добавления/удаления), выбор метода (radio-группа кнопками или чипсами), слайдер/поле для `target_x`, кнопка "Вычислить" (Primary).
- **График** (карточка справа или снизу): занимает основное пространство и заполняет доступную высоту без лишних пустых зон. График (canvas) растягивается по высоте контейнера карточки.
- **История** - раскрывающаяся панель внизу экрана, которая плавно выезжает поверх контента. При большом разрешении может быть закреплена справа в виде боковой панели шириной 320px.

Расположение на десктопе: 2 колонки (панель управления 360px / график занимает оставшееся) с визуальным выравниванием высоты карточек. На мобильных - всё в один столбец с вертикальной прокруткой.

### 6.3 Страница 404

- Центрированное сообщение на всю высоту экрана.
- Крупный заголовок "404" (размер 6rem, цвет `#E88CA5`, вес 800) с декоративными элементами (лёгкие силуэты Рем и Рам в виде векторных контуров, если возможно; иначе просто эмоциональная иконка).
- Текст "Такой страницы не существует. Возможно, она интерполировалась куда-то не туда" (Body Large, цвет `#1E293B`, шутливый тон).
- Кнопка "Вернуться к обучению" (Primary) ведёт на `/learning`.

## 7. Адаптивность и брейкпоинты

Используются стандартные брейкпоинты Vuetify/Bootstrap:

- **xs** (<600px): 1 колонка, отступы 16px, кнопки на всю ширину, график уменьшен, панель управления разворачивается вертикально. Шрифт Body 16px.
- **sm** (600-960px): 1 колонка, но элементы внутри секций могут располагаться горизонтально. Карточки имеют максимальную ширину.
- **md** (960-1264px): возможна двухколоночная раскладка, сайдбар для истории.
- **lg** (1264-1904px): оптимальный вид, контейнер 1200px.
- **xl** (>1904px): контент выровнен по центру, дополнительные отступы.

Все размеры, шрифты и отступы заданы в относительных единицах (`rem`), что упрощает масштабирование.

## 8. Интеграция с Vuetify 3

Для кастомизации темы используем в `frontend/src/plugins/vuetify.js`:

```js
import 'vuetify/styles'
```
